    public class LuceneNGramPFE
extends LuceneFeatureExtractorBase
implements PairFeatureExtractor
    Pair ngram feature extractor for document pair classification. Can be used to extract ngrams from one or both documents in the pair, and parameters for each document (view 1's, view 2's) can be set separately, or both documents can be treated together as one extended document.
    Note that ngram features created by this class are each from a single document, i.e., not combinations of ngrams from the pair of documents. To make combinations of ngrams across both documents, please use LuceneNGramCPFE.

      • PARAM_MARK_VIEWBLIND_NGRAMS_WITH_LOCAL_VIEW

        public static final String PARAM_MARK_VIEWBLIND_NGRAMS_WITH_LOCAL_VIEW
        This option collects a FrequencyDistribution of ngrams across both documents of all pairs, but when writing features, the view where a particular ngram is found is recorded with the ngram. For example, using a #PARAM_NGRAM_USE_TOP_K_ALL value of 500, 400 of the ngrams in the top 500 might happen to be from View 2's; and whenever an ngram from the 500 is seen in any document, view 1 or 2, the document's view is recorded.
        E.g., Feature: view2allNG_Dear
        In order to use this option, PARAM_USE_VIEWBLIND_NGRAMS_AS_FEATURES must also be set to true.

      • PARAM_NGRAM_BINARY_FEATURE_VALUES_COMBO

        public static final String PARAM_NGRAM_BINARY_FEATURE_VALUES_COMBO
        Whether features should be marked with binary (occurs, doesn't occur in this document pair) values, versus the document count of the feature. In combo ngrams this is (doc1freq * doc2freq). Note this only applies to feature values; frequency selection of features is based on frequency across documents, not within documents.
